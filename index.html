MySQL Tips

2016-04-18
database

1.Linux下的MySQL，数据库名、表名、变量名默认是严格区分大小写的。

让MySQL不区分表名大小写：
　　1.用ROOT登录，修改/etc/my.cnf
　　2.在[mysqld]下加入一行：lower_case_table_names=1 （0区分，1不区分）
　　3.重新启动数据库 /etc/init.d/mysqld restart

2.MySQL编码格式：
全部设置成utf8是终极解决方案。

3.MySQL临时表不能在一次查询中被打开两次。会报错Can’t reopen table。解决方法：建个普通表，用完之后再删了。

4.在Linux环境下执行SQL语句要注意tab键问题。因为终端下tab键有提示命令作用，替换成空格之类后再执行。

Java Program with NeticaJ and Cron Job

2016-04-13
tips

同事两年前写的项目，需要部署到服务器上运行。
配置：
1.拷贝64位的Netica文件Netica.dll,NeticaJ.dll,NeticaJ.jar,Netica.lib到java.library.path下
获取java.library.path的方法，在Myeclipse下运行System.out.println(System.getProperty("java.library.path"));
2.修改数据库连接字符串,ODBC连接,文件存储位置

问题：
1.图形化控件各种bug，和算法逻辑没关系，干脆都干掉了
2.服务器系统和数据库经历各种重装，odbc连接各种奇葩问题，原来的连接是这样：
Driver={SQL Server};Server=DB_server;DataBase=myDB
改来改去就是连不上，后来查了手册，看到了角落里面的一行小字

Hint: If you are having difficulty getting your connection string to work, then use the Windows ODBC Data Source Administrator to connect to the database, give it a data source name (DSN), “myName”, and then use “DSN=myName” as the simplified connection string. The Administrator has powerful wizards to facilitate both finding the database and defining precisely how you wish to access it.

于是再配了一遍，改成了这样：
DSN=myDataSource;UID=myUserAccount;PWD=myPassword
大功告成。

然后又提出了一个需求，要做成定时任务。
首先想到的就是Quartz ，以前的程序有过用Quartz+Spring+Tomcat做调度的。但是这是个单独的java程序，没必要搞那么重量级。单独用Quartz 又要考虑到数据库持久化，Quartz 程序的销毁问题。总觉得有更简单的办法。
当当当当，直接写个bat脚本，计划任务调bat，bat调java。

写bat中间最难的问题是程序调用了Netica的dll文件和lib文件，报各种错，比如：
1.java.lang.UnsatisfiedLinkError：no xxx in java.library.path
2.java.lang.ClassLoader.loadClass(Unknown Source)
3.java.lang.UnsatisfiedLinkError: %1 不是有效的 Win32 应用程序

1和2的解决方法是配置好classpath和java.library.path。配置classpath的时候千万要注意在原来的基础上追加，我就手残的把原来的覆盖掉了。
3是因为32位和64位引起的问题。把Netica的x64_bin下的所有文件复制到c:\Program Files\Java\jdkxxx\bin目录下。
设置编译和运行环境的时候注意用64位的目录下的文件和程序，用和操作系统、jdk版本同一个位长的，不然各种报错。

bat文件如果闪退的话可以加pause，退出cmd窗口用exit。
bat的注释是::和rem，<是传递并覆盖，<<是传递并追加。
.表示当前目录，..表示上级目录。

编译
javac -deprecation -classpath ..\x64_bin\NeticaJ.jar;..\x64_bin\sqljdbc4.jar *.java
运行
echo %date%_%time% >> E:\projectname\log\log%date%.txt
set PATH=..\bin;%PATH%
java -classpath .;..\x64_bin\NeticaJ.jar;..\x64_bin\sqljdbc4.jar -Djava.library.path=..\x64_bin Entry >> E:\projectname\log\log%date%.txt
exit
计划任务
然后到计划任务里面进行设置。这里要注意路径问题，计划任务不能识别相对路径，要么配置的时候配置“起始于”选项，要么改成绝对路径。

跨平台 Microsoft SQL Server 迁移到MySQL

2016-04-01
database

1.WINDOWS平台，用MySQL的Workbench工具进行SQL Server数据库迁移
可参阅：Guide to the MySQL Workbench Migration Wizard: From Microsoft SQL Server to MySQL
问题：编码格式，有一个表用的中文做列名，带来很多奇葩问题，简单粗暴地改英文名了。因为原本程序都要改一遍，也没有别的影响。权衡利弊之下，似乎是代价最小的办法。

2.WINDOWS平台，进行MySQL数据库及全部数据导出
数据库结构：mysqldump -u username -p -d dbname > dbname.sql
整个数据库，包括数据：mysqldump -u username -p dbname > dbname.sql

3.LINUX平台，进行MySQL数据库导入
新建数据库 create database dbname
选中该数据库 use dbname
导入数据库 source /home/test/dbname.sql
问题：两个MySQL数据库版本不一致，数据类型略有区别。datetime(6) ，只有MySQL特定版本支持，改成了datetime

Mac下的Hexo安装和配置

2016-03-25
tips

安装homebrew
1.ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"有时候网络不太好，要多试几次。
2.添加环境变量 echo 'export PATH="/usr/local/bin:$PATH"' >> ~/.bash_profile
3.查看安装情况 brew doctor

安装Xcode 非必须
官方文档说可能会碰到一些编译问题，最好装一个。到App Store下载安装，有点大有点慢。

安装Nodejs
brew install node

安装Git
brew install git

配置Git
1.申请github账号
2.建立github仓库，比如username.github.io这种
3.ssh key配置，非必须
a.检查ssh key是否存在 cd ~/.ssh
b.没有的话生成 ssh-keygen -t rsa -C "xxx@xxx.com"
c.在github页面添加ssh key，Account Settings->SSH Public Keys->Add another key，保存
d.测试连接 ssh -T git@github.com
4.个人信息设置 git config --global user.name "username"
git config --global user.email "xxx@xxx.com"

安装Hexo
npm install -g hexo-cli

配置Hexo
blog是文件夹的名字，可以改别的。
hexo init blog
cd blog
npm install
整个网站的配置在_config.yml里面。
配置说明

基本用法
hexo new "new-post"
hexo generate
hexo server
可以访问http://localhost:4000/查看页面了。Ctrl+C停止。
以后generate之前最好运行hexo clean清理缓存文件。
配置了git之后用hexo deploy推送到github上。
其他命令

配置Hexo推送到Git
安装工具 npm install hexo-deployer-git --save
在_config.yml中修改

deploy:
type: git
repo: https://github.com/username/username.github.io.git
branch: master

Linux下的Kettle+MySQL安装配置

2016-03-01
tips

安装JDK
1.查看是否已经安装 rpm -qa |grep java
2.查看有资源的java版本 yum search java
3.用root用户安装对应版本 yum -y install java-1.7.0-openjdk*
4.查看安装效果 java -version

安装MySQL
1.查看是否已经安装 rpm -qa|grep mysql
2.用root用户安装 yum install -y mysql-server mysql mysql-devel
3.查看安装信息 rpm -qi mysql-server
4.启动MySQL进程 service mysqld start

用户设置：
在MySQL启动状态创建管理员 /usr/bin/mysqladmin -u root password '123456'
登陆 mysql -u root -p 输入密码

开机启动配置：
查看MySQL开机选项 chkconfig --list|grep mysqld
设为开机启动 chkconfig mysqld on
检查是否设置成功 chkconfig --list|grep mysqld

其他：
查看配置文件 cat /etc/my.cnf
查看数据库 ll -h /var/lib/mysql
查看日志 ll -h /var/log
查看端口情况 netstat -anp |more

安装Kettle
1.上传data-integration.zip到Linux系统中
2.解压 unzip data-integration.zip -d /home/test/kettle
3.cd到data-integration目录下 ll查看权限
4.加权限 chmod +x *.sh
5.查看安装效果 ./kitchen.sh 显示帮助信息 ./spoon.sh 运行程序，弹出图形界面，安装成功

MySQL连接不上：
1.将MySQL对应mysql-connector-java-5.1.35-bin.jar复制到Kettle对应的libext/JDBC目录下
2.检查数据库连接配置（数据库类型，地址，数据库名称，端口号，用户名，密码）是否正确

RunqianReport Tips

2015-11-01
tips

润乾报表是一个纯JAVA的企业级报表工具，支持对J2EE系统的嵌入式部署，无缝集成。服务器端支持各种常见的操作系统，支持各种常见的关系数据库和各类J2EE的应用服务器，客户端采用标准纯html方式展现。

1.报表要先发布才能部署，不然会报找不到路径或文件不存在的错。

2.select和select1的区别：select1只选一个，select选所有符合条件的。

3.格式控制：加行控制边框，通过if条件控制行数奇偶或值的大小显示颜色。
例如 前景色=if(value()<0,-65536,-16777216)
背景色=if(row()%2==1,-1,rgb(204,255,204))

4.利用隐藏行隐藏列计算中间数据。

5.把选出来的多条记录合并成一条记录
a.通过自定义数据集，用java写，类名为com.runqian.NewDataSetForA之类的
b.通过sql语句（sqlserver2005之后版本支持），利用stuff和for xml path，类似
select a, b = (stuff((select ',' + b from t where t.a = t1.a for xml path('')),1,1,'')) from t1 group by a
c.通过自定义sql函数，类似dbo.f_strUnite
注：c方法只能在数据库中用，不能移植到润乾中。用b方法比a方法要慢。

Project Euler 1-50 Summary

2015-04-01
programming

算法学的一把渣，想了半天，还想到树啊左孩子右孩子什么的，完全忘记了排列组合这个神器。

从上往下是暴力解法，从下往上就是有限的并且逐步减小的过程。

数学不好是硬伤。下面是我在别人的博客上看到的解释：

There are 3 kinds of rational numbers:
Expansion is finite (for instance 1/4 = 0.25): those are numbers of the form (2^a)*(5^b).
Expansion is repetitive, from the start (like 1/7 = 0.(142857)). This is for numbers co-prime to 10.
Expansion is repetitive, after some initial non-repetitive section. This is for numbers that have both factors co-prime to 10 and are divisible by 2 or 5. (for example 1/14 = 0.0(714285)).
To find the longest cycle it’s sufficient to look just at the primes - because other numbers will just have the same cycles after some initial non-repetitive section.

然后模拟求循环节的过程就可以了。

Open Course Recommendation

2014-12-01
other

以下是我之前看的一些名校公开课。间隔时间比较久，突然想起来记录一下。非常主观，仅供参考。

1.Justice: What’s The Right Thing To Do

  • 鼎鼎大名的哈佛公开课。很思辨。所以说美国的那一小撮精英是真的很精英。
  • 难易程度：4
  • 推荐程度：5

2.The Psychology, Biology and Politics of Food

  • 介绍关于食物的心理学、生物学、政治学知识。范围有点广。还是很有意思的。
  • 难易程度：3
  • 推荐程度：4

3.Positive Psychology at Harvard

  • 大名鼎鼎的哈佛幸福课。讲的其实都是很常识的东西。但是常识的东西你不一定能够做到。
  • 难易程度：2
  • 推荐程度：3

4.Financial Markets

  • 耶鲁公开课金融市场。现代人必备常识。讲得很好。
  • 难易程度：3
  • 推荐程度：4

5.Europe Civilization

  • 讲某个时间段内欧洲文明的发展历史。一战死了很多人等等。
  • 难易程度：3
  • 推荐程度：3

6.Introduction to Psychology

  • 心理学导论。现代人必备常识。
  • 难易程度：3
  • 推荐程度：4

7.Foundations of Modern Social Theory

  • 讲现代社会的发展历史和规律。
  • 难易程度：3
  • 推荐程度：4

8.Capitalism Success Crisis and Reform

  • 讲资本主义的发展历史和危机改革一类的。有点意思。有部分历史人物是怎么也绕不过去的。 跟上面那个可以做对照。
  • 难易程度：3
  • 推荐程度：4

9.Intimate Relationships Families and Couples Psychology

  • 讲有关亲密关系的一些心理学知识。用科学的方法研究关系这种东西还是感觉有点怪怪的。
  • 难易程度：3
  • 推荐程度：2

10.Listening to Music

  • 音乐知识。讲的都是比较专业的东西，不太合一般听众的胃口。不过里面的一些曲子都很棒。
  • 难易程度：3
  • 推荐程度：2

11.CS50

  • 哈佛公开课，计算机入门。老师好帅，比较有热情，讲得很好。光看老师就有动力学下去了。
  • 难易程度：3
  • 推荐程度：4

12.Programming Methodology

  • 讲java的。典型的大学教法。可以和udacity的java课程互为补充。
  • 难易程度：3
  • 推荐程度：3

13.Introduction to Computer Science and Programming

  • 编程入门课程。用的python。顺带讲了一点概率论相关的。
  • 难易程度：3
  • 推荐程度：3

14.Understanding the Global Economy

  • 各种不同背景的人的有关全球经济的讲座。对其中的环境和资源问题比较感兴趣。看讲日本的那一集的时候觉得昨天的日本就是今天的中国，中国可以说不什么的还有房地产什么的。
  • 难易程度：3
  • 推荐程度：2

15.Machine Learning

  • 斯坦福公开课，看完之后感觉我的数学还停留在高中水平。高数和概率论数理统计什么的都需要推倒重来。讲到某一些算法的时候有亲切感。
  • 难易程度：5
  • 推荐程度：4

未完待续。
