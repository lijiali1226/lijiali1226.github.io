概念

常识

TPS——Transaction Per Second 每秒事务处理量
IOPS——Input/Output Operations Per Second 每秒进行读写（I/O）操作的次数，衡量随机访问的性能
QPS——Query Per Second 每秒查询数
RT——Response Time 响应时间
吞吐量——Throughput 系统在单位时间内处理请求的数量
TPC——Transaction Processing Performance Council 一系列事务处理和数据库基准测试的规范

SSD固态硬盘（优点：读取速度，抗震能力，功耗，噪音，发热）
HDD机械硬盘（优点：容量，价格，写入次数，数据恢复）

LINUX相关

LINUX IO调度策略：
1) CFQ(Completely Fair Queuing, 完全公平排队)
2) NOOP(电梯式调度程序)
3) Deadline(截止时间调度程序)
4) AS(预料I/O调度程序)

Deadline确保了在一个截止时间内服务请求，这个截止时间是可调整的，而默认读期限短于写期限。这样就防止了写操作因为不能被读取而饿死的现象。Deadline对数据库环境是最好的选择。

cgroup——controller group，将任意进程进行分组化管理的Linux内核功能。
CFS——完全公平调度器
BPS——Bits Per Second 比特率，传输速度

查看当前系统支持的IO调度算法：
dmesg | grep -i scheduler

io scheduler noop registered
io scheduler anticipatory registered
io scheduler deadline registered
io scheduler cfq registered (default)

查看当前系统的I/O调度算法：
cat /sys/block/sda/queue/scheduler

noop anticipatory deadline [cfq]

临时更改I/O调度算法：
echo deadline>/sys/block/sda/queue/scheduler
查看调度算法cat /sys/block/sda/queue/scheduler

noop anticipatory [deadline] cfq

永久更改I/O调度算法：
修改内核引导参数，加入elevator=调度算法
vi /boot/grub/menu.lst
更改：
kernel ... elevator=deadline
重启，查看调度算法cat /sys/block/sda/queue/scheduler

noop anticipatory [deadline] cfq
