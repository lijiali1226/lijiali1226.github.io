MySQL的SQL捕获及分析

2016-05-08
database

前提：最好有线上在跑的SQL列表

常用SQL捕获方法

1.tcpdump
2.查询日志
3.慢日志

日志类别：

tcpdump

yum install tcpdump
tcpdump -i eth0 -s 0 -l -w - dst port 3306|strings
问题：可能乱码，对人类阅读不友好
改进：用pt-query-digest分析tcpdump的抓包

查询日志

show global variables like "general%";
启用查询日志：
set global general_log=1;

慢日志

记录执行时间超过long_query_time的sql
show global variables like "slow%";
启用慢日志需要修改配置/etc/my.cnf
在[mysqld]下面加入：
slow_query_log=TRUE
long_query_time=3
slow_query_log_file=/var/lib/mysql/slow.log
修改慢日志时间阈值：
set global long_query_time=1;

分析捕获的SQL

pt-query-digest

官网说明
文件概述
profile信息
具体的sql

文件概述：分析的慢日志名，sql total，sql unique，每秒慢日志数量QPS，并发度等
profile：
Rank 默认按查询时间降序排列
Responestime 响应时间，占所有响应时间的百分比
Calls 查询执行次数
R/Call 平均响应时间

安装
wget https://www.percona.com/downloads/percona-toolkit/2.2.18/RPM/percona-toolkit-2.2.18-1.noarch.rpm
yum install perl-Time-HiRes perl-IO-Socket-SSL perl-TermReadKey
rpm -ivh percona-toolkit-2.2.18-1.noarch.rpm

慢日志：
pt-query-digest slow.log
用show processlist 获取：
pt-query-digest -processlist h=host1
用tcpdump获取：
tcpdump -s 65535 -x -nn -q -tttt -i any -c 1000 port 3306>mysql.tcp.txt
pt-query-digest --type tcpdump mysql.tcp.txt

分析重点：数量最多的，并发度高的，占用时间多的

基于平台管理MySQL日志：Anemometer（github）
