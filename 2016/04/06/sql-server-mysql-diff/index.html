Microsoft SQL Server 和MySQL的区别（数据类型及语法）

2016-04-06
database

数据类型
两者大部分数据类型都能对应，少部分需进行转换
(图片来自MySQL)

日期函数
(图片来自MySQL)

举例：
select date_format(date_sub('2016-05-01',interval 1 day),'%Y%m%d')
结果：
20160430

select date_format(date_sub(a,interval 21 day),'%Y%m%d') from dbname.tablename
column a (varchar)
20160121
20160122
20160123

结果：
20151231
20160101
20160102

日期转字符串：
DATE_FORMAT(date,format)
格式控制

字符串转日期：
STR_TO_DATE(str,format)

之前的同事设计的数据库把一个非常重要的日期字段在所有的表中都用的varchar类型存储，带来了无数的日期字符串转换问题，真是一个非常糟糕的设计。

类型转换
SQL Server
CONVERT(int, '123.4')
CAST('123.4' as int)
convert可以格式化日期和数值。

MySQL
CONVERT('23',SIGNED)
CAST('125e342.83' AS signed)
可以转换的类型是有限制的。

临时表
SQL Server
select * INTO #TEMPOO FROM SELECT * FROM table_name

MySQL
CREATE TEMPORARY TABLE tmp_table SELECT * FROM table_name
