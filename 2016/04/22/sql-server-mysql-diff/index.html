Microsoft SQL Server 和MySQL的区别（数据类型及语法）

2016-04-22
database

数据类型

两者大部分数据类型都能对应，少部分需进行转换
(图片来自MySQL)

日期函数

(图片来自MySQL)

日期转字符串：
DATE_FORMAT(date,format)
格式控制

字符串转日期：
STR_TO_DATE(str,format)

之前的同事设计的数据库把一个非常重要的日期字段在所有的表中都用的varchar类型存储，带来了无数的日期字符串转换问题，真是一个非常糟糕的设计。

类型转换

SQL Server
CONVERT(int, '123.4')
CAST('123.4' as int)
convert可以格式化日期和数值。

MySQL
CONVERT('23',SIGNED)
CAST('125e342.83' AS signed)
可以转换的类型是有限制的。

临时表

SQL Server
select * INTO #TEMPOO FROM SELECT * FROM table_name

#局部临时表 ##全局临时表 @本地变量 @@函数 N以Unicode格式存储

MySQL
CREATE TEMPORARY TABLE tmp_table SELECT * FROM table_name

字符串

SQL Server用+连接字符串，MySQL用concat()函数

其它语法

SQL Server
select top 100 * from table

MySQL
select * from table limit 100
