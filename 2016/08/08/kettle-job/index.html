Java中调用Kettle Job报错

2016-08-08
data

背景：
Kettle4.4版本。报错：
Could not read from “jar:file:///D:/kpi.jar!/com/yag/ETL/yunshushengchan/parsetime.ktr” because it is a not a file

Eclipse运行时均正常，但将程序打jar包后直接运行就报错。其中，job的路径为相对路径，
java.net.URL Url = getClass().getResource("ETL/gukeyijian/");
String tmp = Url.toString();
RunJob.runJob(null, null, tmp + "gkyj.kjb");
job的实际路径如下 jar:file:/D:/kpi.jar!/com/yag/ETL/yunshushengchan/gkyj.kjb，是能被读出来的正确文件路径。

trans在job中被调用，通过job中的变量${Internal.Job.Filename.Directory}来确定路径，预览kettle的路径如file:///D:/Users/workspace/kpi/src/com/yag/ETL/yunshushengchan/parsetime.ktr。在jar包中调用的时候路径就成了jar:file:///D:/kpi.jar!/com/yag/ETL/yunshushengchan/parsetime.ktr。这种路径在kettle直接运行的时候是可以的，但是通过jar包运行就识别不了，
jar包运行ktr正确文件路径应该如jar:file:/D:/kpi.jar!/com/yag/ETL/yunshushengchan/parsetime.ktr。

解决方法：
容易想到的办法就是拆分job，job里面只封装如ftp文件下载，本地文件删除之类的方法，trans直接用java程序通过相对路径调用，就可以避免jar包运行时的路径问题。前提是job的逻辑不是太复杂。
但是这样有个致命的问题，就是编译之后的代码执行顺序无法确定，job肯定是有时序的，而java不知道运行的是哪一个trans。

最终的解决办法还是用job调用trans，但是不用kettle的变量，通过传自定义参数控制路径。
