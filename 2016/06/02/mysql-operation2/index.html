测试工具

fio

官网

安装

yum install libaio libaio-devel
wget http://brick.kernel.dk/snaps/fio-2.1.10.tar.gz
tar -xzvf fio-2.1.10.tar.gz
cd fio-2.1.10
./configure
make
make install

读写测试

随机读：
fio -filename=/dev/sdb1 -direct=1 -iodepth 1 -thread -rw=randread -ioengine=psync -bs=16k -size=5G -numjobs=10 -runtime=1000 -group_reporting -name=mytest

虚拟机测试结果：
read:io=51200MB, bw=412140KB/s, iops=25758, runt=127211msec
（省略其他输出）

随机写：
fio -filename=/dev/sdb1 -direct=1 -iodepth 1 -thread -rw=randwrite -ioengine=psync -bs=16k -size=5G -numjobs=30 -runtime=1000 -group_reporting -name=mytest

虚拟机测试结果：
write: io=1112.5MB, bw=1138.6KB/s, iops=71, runt=1000204msec
（省略其他输出）

混合随机读写：
fio -filename=/dev/sdb1 -direct=1 -iodepth 1 -thread -rw=randrw -rwmixread=70 -ioengine=psync -bs=16k -size=200G -numjobs=30 -runtime=100 -group_reporting -name=mytest -ioscheduler=noop

虚拟机测试结果：
read: io=347312KB, bw=3469.3KB/s, iops=216, runt=100113msec
write: io=144736KB, bw=1445.8KB/s, iops=90, runt=100113msec
（省略其他输出）

dd（Linux命令）

作用：用指定大小的块拷贝一个文件

写速度：
time dd if=/dev/zero of=/test.dbf bs=8k count=100000

读速度：
time dd if=/dev/sdb1 of=/dev/null bs=8k

读写速度：
time dd if=/dev/sdb1 of=/test.dbf bs=64k

/dev/zero：产生字符
if=/dev/zero不产生IO，因此可以用来测试写速度。
/dev/null：回收站
of=/dev/null不产生IO，可以用来测试读速度。
time有计时作用，dd用于复制，从if读出，写到of。
bs是每次读或写的大小，即块的大小，count是读写块的数量。

tpcc-mysql

官网

安装

一开始直接用的yum install bzr，版本太老，python用的2.6的，装tpcc-mysql和sysbench各种报错，于是就自己下了新版，重新装了一遍。
wget https://launchpad.net/bzr/2.7/2.7.0/+download/bzr-2.7.0.tar.gz
tar -xzvf bzr-2.7.0.tar.gz
yum install Pyrex
python setup.py install
bzr branch lp:~percona-dev/perconatools/tpcc-mysql
cd tpcc-mysql/src
make

测试

tpcc_load 初始化数据
tpcc_start 进行压力测试
–help查看用法
注意$PATH中要有mysql_config
建测试库 mysqladmin -uroot -p create tpcc10
建表 mysql -uroot -p tpcc10 < create_table.sql
加索引和外键 mysql -uroot -p tpcc10 < add_fkey_idx.sql
初始化数据，最后一个参数是仓库数目 ./tpcc_load 127.0.0.1:3306 tpcc10 root "password" 10
如果要并行加载数据用load.sh
基准测试 ./tpcc_start -h127.0.0.1 -P3306 -dtpcc10 -uroot -p"password" -w10 -c32 -r120 -l1800
-w warehouses：仓库的数量
-c connections：线程数，默认为1
-r warmup_time：热身时间，单位s，默认为10s，热身是为了将数据加载到内存。
-l running_time：测试时间，单位s，默认为20s

虚拟机测试结果：

新订单业务和库存仓储业务均存在延迟，响应耗时指标没有达到标准。
