Java FTP上传下载问题

2016-07-08
tech

问题1：

从FTP下载文件，可以正常下载，但是打开的时候报“excel已损坏”之类的错误。

解决方法：
改编码格式。ftpClient.setControlEncoding("GBK");

问题2：

通过FTP上传文件，本地测试一切正常。服务器上可以正常登陆FTP，通过FTP地址直接上传文件。服务器程序上传测试，FTP里面可以创建文件，但是大小为0，传输不过去，刷新就没有文件了，而上传页面就卡在上传的那一步。

解决方法：
改端口设置。ftpClient.enterLocalPassiveMode();

原因：
FTP通常用的端口是21（命令端口）和20（数据端口）。但FTP工作方式不同，有主动模式（PORT）和被动模式（PASV），数据端口并不总是20。

主动模式：客户端向服务器的FTP端口（默认是21）发送连接请求，服务器接受连接，建立一条命令链路。当需要传送数据时，客户端用PORT命令告诉服务器：我打开了X端口。于是服务器从20端口向客户端的X端口发送连接请求，建立一条数据链路来传送数据。

被动模式：客户端向服务器的FTP端口（默认是21）发送连接请求，服务器接受连接，建立一条命令链路。当需要传送数据时，服务器用PASV命令告诉客户端：我打开了X端口。于是客户端向服务器的X端口发送连接请求，建立一条数据链路来传送数据。

其他注意事项：

上传的文件不要用中文名。
测试的时候一步步来，注意排除干扰项，比如防火墙、杀毒软件之类。
